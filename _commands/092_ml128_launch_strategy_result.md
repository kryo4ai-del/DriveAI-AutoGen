# 092 Launch Strategy — Ergebnis

**Datum**: 2026-03-18
**Ausgefuehrt von**: Claude Code (Mac, Xcode 26.3)

## 1. Target Audience

**Primär**: Fahrschüler in Deutschland, 17-25 Jahre
- Lernen für die theoretische Führerscheinprüfung (Klasse B)
- Haben ein Smartphone (iPhone), nutzen es täglich
- Wollen effizient lernen, nicht stundenlang Bücher wälzen

**Sekundär**: Fahrschulen als Empfehlung
- Fahrlehrer die ihren Schülern eine Lern-App empfehlen

## 2. Distribution — Erste Nutzer

### Phase 1: Soft Launch (0-100 Nutzer)
- **TestFlight**: 20-50 Beta-Tester aus persönlichem Netzwerk
- **Fahrschul-Kontakt**: 2-3 lokale Fahrschulen direkt ansprechen
- **Ziel**: Echtes Feedback, Bugs finden, Fragen-Qualität prüfen

### Phase 2: App Store Launch
- **Kostenlos** (v1) — keine Bezahl-Barriere
- **ASO (App Store Optimization)**: Keywords aus APP_STORE_METADATA.md
- **Organische Suche**: "Führerschein", "Fahrschule", "Theorie" — hohe Suchvolumen

### Phase 3: Wachstum (optional)
- TikTok/Instagram: Kurze "Wusstest du?"-Videos mit Führerschein-Fragen
- Reddit/Gutefrage: Antworten auf Führerschein-Fragen mit App-Hinweis

## 3. Feedback Loop

| Kanal | Zweck |
|---|---|
| App Store Reviews | Öffentliches Feedback, Rating |
| TestFlight Feedback | Beta-Bugs, UX-Probleme |
| E-Mail (Support) | Direkte Fragen, Feature-Requests |
| In-App (v2) | "War diese Frage hilfreich?" |

### Was gemessen werden sollte (v2)
- Welche Fragen werden am häufigsten falsch beantwortet?
- Wie viele Sessions pro User pro Woche?
- Wie lange dauert es bis "Prüfungsbereit"?
- Abbruch-Punkte in der Training-Session

## 4. Realistic Timeline

| Woche | Aktion |
|---|---|
| 1 | App Icon, Developer Account, Privacy URL |
| 2 | TestFlight Build, 20 Beta-Tester |
| 3 | Feedback einarbeiten, Fragen auf 300+ |
| 4 | App Store Submit |
| 5+ | Organisches Wachstum, ASO optimieren |
