# Design Vision & UX Innovation (Kapitel 4.5)

## Zweck
Definiert wie sich die App ANFUEHLT und was sie visuell UNTERSCHEIDET.
Verhindert dass die Factory austauschbare Standard-Apps produziert.

> "Eine App ist nur so gut wie ihr Design — egal wie gut das Produkt dahinter ist."

## Agents
| # | Agent | Aufgabe | Web-Recherche | Modell |
|---|---|---|---|---|
| 17a | Design-Trend-Breaker | Genre-Standard analysieren + Innovationen finden | Ja | Sonnet |
| 17b | UX-Emotion-Architect | Emotionen + Interaktionen + Wow-Momente definieren | Ja | Sonnet |
| 17c | Design-Vision-Compiler | Verbindliches Design-Dokument erstellen | Nein | Sonnet |

## Flow
```
14 Reports -> 17a (was ist Standard, was ist anders) -> 17b (wie fuehlt es sich an) -> 17c (verbindliches Dokument)
```

## Output
Das Design-Vision-Dokument ist VERBINDLICH fuer:
- Kapitel 5 (Asset Audit — Stil-Guide und Consistency-Check basieren darauf)
- Kapitel 6 (Roadbook — Design-Vorgaben werden ins Roadbook uebernommen)
- Produktionslinie (Code-Gen-Agents bekommen das Design-Briefing als Kontext)
